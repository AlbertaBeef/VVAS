VVAS GStreamer Plug-ins for Data Center Platforms¶

The following section describes GStreamer plugins for Data center platforms. The plug-ins source code is available in the ivas-gst-plugins repository/folder of the VVAS sources tree. The following table lists the GStreamer plug-ins.

Table 1: GStreamer Plug-ins

Plug-in Name

Functionality

ivas_xvcudec

GStreamer plug-in to perform hardware accelerated h264/h265 video decoding.

ivas_xvcuenc

GStreamer plug-in to perform hardware accelerated h264/h265 video encoding.

ivas_xlookahead

GStreamer plug-in to perform hardware accelerated h264/h265 video encoding.

Custom Plug-ins¶

This section describes several custom plug-ins for Data Center platforms only. For other GStreamer plugins that can be used for Data Center as well as Embedded platforms are covered in << Link >>

VCU Decoder Plug-in¶

“ivas_xvcudec” is GStreamer Plugin for hardware accelerated H264/H265 Video decoding using VCU. “ivas_xvcudec” plug-in does decoding of H264/H265 streams using VCU decoder soft-kernel running on ARM connected to host via PCIe. This plug-in accepts input encoded stream in byte-stream/NALU format only and produces NV12 frames based VCU frame alignment requirements. To achieve zero-copy for higher performance, decoder plug-in does not bring data from device to host. Instead, it enables flag IVAS_SYNC_FROM_DEVICE on GstIvasMemory which holds raw NV12 frame. In a given GStreamer pipeline, if any of the following element (e.g. filesink) after VCU decoder would like to read and/or write on decoded frame, then DMA copy will be triggered by IVAS allocator.
